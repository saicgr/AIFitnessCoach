## LinkedIn Post (FULL THREAD verdict)

[add image here: Android Authority headline screenshot above the fold — same as X tweet 1; or a side-by-side "What Zealova sees / What Google Health sees" comparison]

Google built the same AI model I use. They launched it today at $9.99 a month. And within 24 hours of the first hands-on reviews, it had fabricated a workout that never happened.

I want to be precise here because this matters for anyone building with AI.

Google Health Coach is powered by Gemini. So is Zealova, the solo AI fitness coach I have been building for the past year. Same model. Completely different outcome.

The difference is not the model. The difference is what the model is handed.

Google Health infers workout sessions from wearable signals: step count, heart rate, GPS traces, sleep patterns. When those signals look like a 5-mile run, the model reports a 5-mile run. There is no ground truth check against what the user actually chose to log. The model is reasoning under uncertainty and presenting the output as fact.

Zealova does not infer. The AI reads what is in the database. If you did not log a session, the session does not exist in the context window. The coach cannot reference something it was never given.

This is a data pipeline design decision, not a model quality decision.

The lesson I take from today: the reliability of an AI feature is set before the first prompt is written. It is set by what data you allow into the context, what data you exclude, and whether you verify inputs against what the user actually chose to record. A state-of-the-art model on unverified wearable inferences will still hallucinate. A simpler model on clean, user-confirmed data will not.

I do not say any of this to pile on. Building a product that processes millions of users' health data is genuinely hard. But today felt like a useful reminder of where the real work is.

What is your mental model for when to trust an AI output in a domain where errors have real consequences?
