As a UX expert + premium user — would I want the "+"?

Honestly: no. I'd drop it.

1 · It's redundant. Home already has the 6-icon quick row (Food, Water, Scan Menu, Fridge…), per-tile log buttons (Water "+", Fasting ▶), and Chat for AI logging. The "+" sheet just re-lists Food/Water/Weight — a second door to the same rooms. Good UX is one obvious path per action; multiple paths read as indecision.

2 · It fights the hierarchy. A bright FAB is a second big purple circle competing with the workout's ▶ START — it visually shouts "this is the main action," but it isn't. The home's job is glance + launch; START should own the spotlight.

3 · Wrong pattern for a dashboard. A FAB earns its place in create-one-thing apps (compose mail, new note). MyFitnessPal's centre "+" works because logging food is literally their whole product. Zealova is broader — logging isn't the single core loop, so an omni-"+" is pattern padding.

4 · Premium ≠ more buttons. A premium user notices restraint. A calm, focused home feels more premium than one with an extra floating control.

Nothing is lost by cutting it: Food/Water = quick actions · Weight/Measure/Mood/ quick-workout-log = behind "More" · natural-language "2 eggs and a coffee" logging = Chat (quick action #1). The valuable kernel — AI logging — already lives in Chat.

The mockup below is the home with no "+". Calmer, one clear hero (START), every logging path still one or two taps away. This is the version I'd ship.
