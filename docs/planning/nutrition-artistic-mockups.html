3 · How it actually ships in Flutter

For true state reactivity (hungry → goal → overstuffed driven by live calories), the production build uses Rive, not CSS:

• Illustrator draws the Buddy/Bowl → animator rigs a State Machine in Rive with an input like fullness (0–1.3) and a justAte trigger.

• Flutter adds the rive package; the home card hosts a RiveAnimation and pushes fullness = eaten/goal from the nutrition provider — the character reacts in real time.

• Lottie is the lighter alt (package lottie) if we only need looping states, no interactive rig.
