Addendum · motion / “intro video” question + 2026 research
Should the intro demo loop appear here? → Yes, as the reel’s beat cards. No, as a background.
The “intro video” is actually the programmatic 4-scene demo loop (intro_demo/demo_scenes.dart) — clock-driven Flutter widgets, no video asset. RevenueCat’s video-paywall data says motion converts best in fitness apps when it’s a UI feature demo, 15–30s, silent, looping — which is exactly what those scenes are. One fitness app reported ~80% revenue lift from paywall video; another doubled install→trial by adding a feature demo to the paywall.

Recommendation: swap the reel’s static mock cards for the LIVE intro scenes (FoodScanScene, ProgramBuilderScene, MenuAnalysisScene) framed inside the beat card — the paywall’s “video” with zero bandwidth/quality risk, zero new assets, and recall continuity from first open. Keep the offer page static (pricing needs calm legibility — motion behind numbers hurts) and the commit pact static (it’s a vow, not a demo).
2026 paywall research checkboxes (and one warning)
✓ Yearly-as-hero + savings badge (highest LTV default) — in the AFTER.
✓ Transparency strip: cancel terms / reminder / $0-today visible, not buried — in the AFTER.
✓ Day-0 placement after demonstrated value (82% of trials start day 0) — funnel already does this.
✓ Comparison table vs single-job apps — the features screen already carries it.
