CHANGE 2 OF 4

Home hero simplification — one time-aware answer to "what now?"

✕ REJECTED 2026-06-11 — Home stays as-is. You were right; the code re-review confirms it.

Why this was the wrong proposal (verified in code): (1) StackedBannerPanel already collapses to one visible banner with the rest layered behind, has swipe-to-dismiss-reveal-next AND a dismiss-all (X badge → pill) — it uses less space than the proposed banner + actions row. (2) The coach hero card already IS the time-aware "what now" element: Gemini daily insight, whole-card tap opens chat prefilled, primary/secondary CTAs come from the API payload, long-press regenerates, minimize/dismiss-today. A client-side time-of-day hero would duplicate it with a dumber version. (3) The default order was deliberately tuned across v27–v34 — "deck glance → quick actions → coach framing → workout act" was an explicit user request in v34. Frames below kept dimmed for the record.

Entrance staggers (nutrition pills) compose from standard + per-item delay rather than a bespoke 1200ms controller. Count-up numbers keep their 900–1000ms tween (data, not chrome).

On the word bounce: yes, but with two rules. (1) One element per event — when a nav tab is selected, either the icon spin-pops or the label bounces, never both at once; the bounce fits best on value changes (streak ticking 12→13, XP landing, kcal goal hit) where it draws the eye to what changed. (2) Single bounce, no loop, respects the reduced-motion accessibility setting. Used that way it adds life without becoming noise.
